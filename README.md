# 🏭 設備稼働率ダッシュボード

PythonとStreamlitを使用して作成した、設備稼働率の可視化アプリです。  
GitHub Actionsによる定期データ生成と、Streamlit CloudによるWeb公開を行っています。

## 🌐 アプリURL

https://equipment-operation-dashboard-midkcrlr9bksdimu9be5dd.streamlit.app/

---

## 📋 概要

製造業では、設備の稼働状況や停止理由を把握し、改善活動につなげることが重要です。

本アプリでは、設備ごとの稼働率・停止時間・停止理由を集計し、日別・週別・ライン別・設備別に可視化できるようにしました。

---

## 🚀 主な機能

### 稼働率の可視化

- 設備別の日別稼働率推移
- ライン別の日別稼働率推移
- 設備別の週別稼働率推移
- ライン別の週別稼働率推移
- 実績稼働率と想定稼働率の比較

### 停止情報の分析

- 総停止時間の表示
- 停止理由ランキング
- 停止理由別の停止時間グラフ
- 想定稼働率を下回った設備のアラート表示

### データ生成の自動化

GitHub Actionsにより、以下の時間帯で定期的にダミーデータを生成します。

- 0:00
- 6:00
- 12:00
- 18:00

また、日曜日・日本の祝日は休業日としてデータ生成をスキップします。  
休業日明けは8:30始業として計画時間を自動調整しています。

## ⚙ GitHub Actions

設備稼働率データは、GitHub Actionsにより定期生成されています。

データ生成時には以下を考慮しています。

- 6時間ごとのデータ生成
- 日曜日の休業判定
- 日本の祝日判定
- 休業日明けの8:30始業
- 月ごとのCSVファイル分割
- Streamlit Cloudでの自動反映

---

## 🔐 データ管理について

本アプリで使用しているデータは、実在の企業・設備・生産情報とは関係のないダミーデータです。

当初はDBでの管理も検討しましたが、公開ポートフォリオとして安全に運用するため、接続情報や認証情報を扱わないCSV管理を採用しています。

## 👨‍💻 開発背景

現在、製造業向けDX業務に携わっており、設備稼働率・生産実績・停止理由などのデータ活用に関心があります。

業務理解を深めるため、個人開発として設備稼働率ダッシュボードを作成しました。

単なるグラフ表示だけでなく、GitHub Actionsによる定期データ生成、休日判定、休業日明けの計画時間調整など、実運用を想定した仕組みを取り入れています。
